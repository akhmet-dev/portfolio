# №3 Практикалық жұмыс: «Менің портфолиом» веб-сайтының құрылымын құру және жүзеге асыру

**Тақырыбы:** Өзі құрған макет бойынша HTML және CSS қолданып, «Менің портфолиом» веб-сайтының құрылымын құру және жобаны жүзеге асыру  
**Орындаған:** Сарсенбай Ахмет  
**Сілтеме:** [portfoliosarsenbayakhmet.vercel.app](https://portfoliosarsenbayakhmet.vercel.app/)

---

## 1. Жобаның файлдық құрылымы (2-кезең)
Жоба келесі файлдық құрылымға сәйкес ұйымдастырылған:
```text
portfolio/
├── index.html        # Веб-сайттың негізгі құрылымы (HTML5)
├── style.css         # Сайттың стилі және адаптивті дизайны (CSS3)
├── script.js        # Интерактивті қосымша мүмкіндіктер (JavaScript)
└── images/           # Жобада пайдаланылатын суреттер папкасы (суреттер орналасатын орын)
```

## 2. Веб-сайттың негізгі бөлімдері (3, 4, 5-кезең)
1. **Навигациялық мәзір (Header):** `akhmet.dev` логотипі, Басты бет, Мен туралы, Дағдылар, Жобалар, Байланыс сілтемелері және Түйіндеме батырмасы.
2. **Басты бет (Home/Hero):** Жеке аватар суреті, белсенді студент статусы, өзін таныстыру мәтіні және таңдаулы жобалар/ұйымдардың (`Janarym`, `FoodSave`, `Enactus AITUC`) белгішелері.
3. **Мен туралы (About Me):** Студенттің Enactus командасындағы рөлі, басқарушылық және ұйымдастырушылық қабілеттері, және бұрышпен сәл бұрылып тұратын стильді фотокартасы.
4. **Техникалық Дағдылар (Skills):** Дағдыларды таңдауға арналған интерактивті батырмалар және таңдалған дағды туралы толық мәлімет көрсететін динамикалық карточка.
5. **Таңдаулы Жобалар (Projects):** Жобаларды санаттар бойынша («Барлығы», «Стартаптар», «Enactus») сүзгіден (filter) өткізу мүмкіндігі бар интерактивті сүзгі және жоба карточкалары.
6. **Байланыс (Contact):** Telegram, Instagram, GitHub және электрондық поштаға арналған сілтемелік карточкалар (электрондық поштаны көшіру батырмасы жұмыс істейді).
7. **Footer бөлімі:** Логотип, әлеуметтік желі сілтемелері және авторлық құқық туралы ақпарат.

## 3. Бақылау сұрақтарына жауаптар

### 1. HTML қандай қызмет атқарады?
HTML (HyperText Markup Language) — веб-беттерді құруға арналған стандартты белгілеу тілі. Ол веб-сайттың қаңқасын (құрылымын) анықтайды. Мысалы, мәтінді, суреттерді, кестелерді, формаларды және сілтемелерді бетте орналастыру үшін қолданылады.

### 2. CSS не үшін қолданылады?
CSS (Cascading Style Sheets) — HTML құжаттарының сыртқы көрінісін (стилін) безендіруге арналған каскадты стильдер кестесі. Ол қаріптерді, түстерді, арақашықтықтарды, блоктардың орналасуын, анимацияларды, hover (тышқанды үстіне апарғандағы) эффектілерді және адаптивтілікті реттейді.

### 3. Портфолио веб-сайты дегеніміз не?
Портфолио веб-сайты — маманның немесе студенттің жеке ақпараттарын, білімін, жұмыс тәжірибесін, дағдыларын, жасаған жобаларын және байланыс деректерін көрсететін жеке веб-ресурсы. Ол жұмыс берушілерге немесе оқытушыларға өз қабілеттерін визуальды және кәсіби түрде көрсетуге көмектеседі.

### 4. Header элементінің қызметі қандай?
`<header>` — веб-беттің немесе бөлімнің кіріспе бөлігін (жоғарғы деректемесін) білдіретін семантикалық HTML элементі. Әдетте мұнда сайт логотипі, навигациялық мәзір, іздеу жолағы және пайдаланушы профилі орналасады.

### 5. Navigation мәзірі не үшін қажет?
Navigation мәзірі (әдетте `<nav>` тегі ішінде) пайдаланушыға веб-сайттың әртүрлі бөлімдері немесе беттері арасында жылдам әрі ыңғайлы ауысып жүруге мүмкіндік береді. Ол сайттың пайдаланушылық тәжірибесін (UX) жақсартады.

### 6. Footer бөлімінде қандай ақпарат орналастырылады?
`<footer>` — беттің немесе бөлімнің ең төменгі жағында орналасатын бөлік. Онда авторлық құқықтар туралы ақпарат (Copyright), әлеуметтік желі сілтемелері, байланыс нөмірлері/пошталары, пайдалану шарттары мен құпиялылық саясатына сілтемелер жазылады.

### 7. Адаптивті дизайн дегеніміз не?
Адаптивті дизайн (Responsive Design) — веб-сайттың кез келген құрылғының (компьютер, планшет, смартфон) экран өлшеміне бейімделіп, дұрыс және ыңғайлы көрсетілуін қамтамасыз ететін әзірлеу әдісі. Ол CSS медиа сұраныстары (Media Queries, `@media`) арқылы жүзеге асырылады.

### 8. Макет пен дайын сайттың сәйкестігі не үшін маңызды?
Макет пен дайын сайттың сәйкестігі (Pixel Perfect) веб-дизайнер мен веб-әзірлеуші арасындағы келісілген концепцияның бұзылмауын қамтамасыз етеді. Сәйкестік кәсібилікті білдіреді және пайдаланушы интерфейсінің (UI) жоспарланған эстетикасы мен ыңғайлылығын (UX) дәл жеткізу үшін өте маңызды.

### 9. CSS арқылы қандай дизайн элементтерін өзгертуге болады?
CSS арқылы келесі элементтерді өзгертуге болады:
* Мәтіннің және беттің түсін (`color`, `background-color`)
* Қаріптерді (түрі, өлшемі, қалыңдығы: `font-family`, `font-size`, `font-weight`)
* Шегіністер мен өлшемдерді (`margin`, `padding`, `width`, `height`)
* Блоктардың орналасу стилін (`display: flex`, `display: grid`, `position`)
* Бордюрлер мен бұрыштарды жұмырлауды (`border`, `border-radius`)
* Ауысу эффектілері мен анимацияларды (`transition`, `animation`)

### 10. Веб-сайт құрылымын жоспарлаудың артықшылығы қандай?
Веб-сайт құрылымын алдын ала жоспарлау (макет жасау):
* Код жазу кезінде уақыт үнемдеуге мүмкіндік береді.
* Дизайндағы қателіктер мен логикалық кемшіліктерді код жазбай тұрып түзетуге көмектеседі.
* Семантикалық тұрғыдан дұрыс, іздеу жүйелеріне қолайлы (SEO) және таза код жазуға жағдай жасайды.
* Пайдаланушы үшін сайт навигациясының түсінікті және ыңғайлы болуын қамтамасыз етеді.
